HW3 — LLM Tool Use & ReAct Agents

Released: March 23  ·  Due: April 6, 11:59pm  ·  Points: 100

Overview

You will build a ReAct-style agent (Reason + Act) that uses an LLM backbone combined with a set of tools to answer multi-hop questions. This is the core architecture behind many production agentic systems today.

Learning Objectives

  • Implement the ReAct agent loop (thought → action → observation → repeat).
  • Define and register tools using the OpenAI or Anthropic function-calling API.
  • Evaluate agent accuracy on a benchmark of multi-hop questions.
  • Analyze failure modes: hallucination, tool misuse, infinite loops.

Provided Tools

  • wikipedia_search(query) — Returns the first paragraph of a Wikipedia article.
  • calculator(expression) — Evaluates a safe arithmetic expression.
  • get_date() — Returns today's date.

Tasks

  1. Implement ReActAgent (40 pts) — Complete the agent loop in agents/react_agent.py. The agent must parse LLM output to detect tool calls, execute them, and feed results back.
  2. Benchmark Evaluation (30 pts) — Run your agent on 50 provided questions from data/questions.json. Compute exact-match accuracy and average number of tool calls per question.
  3. Prompt Engineering (15 pts) — Experiment with at least two different system-prompt designs. Report how accuracy changes and explain why.
  4. Error Analysis (15 pts) — Identify and categorize 5 failure cases. What went wrong and how could it be fixed?

API Usage & Cost

Use GPT-4o-mini or Claude Haiku to keep costs low during development. The benchmark evaluation should cost under $1 total. Do not use your key on the full benchmark more than once — batch your calls.

Reading

Yao et al. (2023), "ReAct: Synergizing Reasoning and Acting in Language Models." ICLR 2023. (Posted on Moodle.)
