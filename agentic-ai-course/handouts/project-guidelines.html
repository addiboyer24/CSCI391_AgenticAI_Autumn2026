Final Project Guidelines

Overview

The final project is an open-ended opportunity to design, implement, and evaluate an agentic AI system of your choosing. Projects may be done individually or in groups of up to three.

Deliverables & Timeline

  • Proposal (Apr 1) — 1–2 page PDF describing your agent, task domain, planned evaluation, and team members. Submit via Moodle.
  • Presentation (Apr 22 – May 1) — 12-minute live demo + slides. Schedule posted mid-April.
  • Final Report (May 5) — 6–10 page written report (see format below). Code submitted as a GitHub repo link.

Project Ideas

  • An LLM agent that autonomously browses the web and answers research questions with citations.
  • A multi-agent debate system where two agents argue positions and a judge evaluates quality.
  • A planning agent that manages a simulated household (grocery lists, scheduling, etc.).
  • A reinforcement learning agent trained on a custom Gymnasium environment.
  • A code-generation agent with iterative testing and self-correction.
  • An embodied agent in simulation (e.g., MiniGrid, PyBullet).

Report Format

  1. Introduction — Motivation, problem statement, contributions.
  2. Related Work — At least 5 relevant papers or systems.
  3. System Design — Architecture diagram, agent loop, tools/actions, knowledge representation.
  4. Experiments & Evaluation — Metrics, baselines, results tables or figures.
  5. Discussion — What worked, what didn't, failure modes, limitations.
  6. Conclusion & Future Work

Grading Rubric (30% of course grade)

  • Technical depth (40%) — Non-trivial implementation, thoughtful design choices.
  • Evaluation quality (25%) — Clear metrics, meaningful baselines, honest analysis.
  • Presentation (20%) — Clear demo, articulate explanation of design and results.
  • Report writing (15%) — Clarity, completeness, proper citations.
